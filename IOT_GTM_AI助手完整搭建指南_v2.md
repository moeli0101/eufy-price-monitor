#### 什么是Vibe Coding？

**不是写需求文档，而是"vibe"对话**：

| 传统编程 | Vibe Coding |
|---------|------------|
| "需求文档第3.2.1条..." | "好乱" |
| "技术方案评审会..." | "太复杂了" |
| "测试用例47条..." | "我躺平了" |

**关键对话示例**：
```
你："好乱" （2个字）
CC：理解 = UI需要重构 → 重新设计整个界面

你："我躺平了" （情绪表达）
CC：理解 = 需要零维护 → 做到完全自动化

你："我真的要生气了"
CC：理解 = 问题很严重 → 优先级最高，立即修复
```

**Vibe Coding的本质**：
> "不是写代码，而是vibe对了，代码自然就对了"

cd ~/Desktop/feishu_bot

## 4️⃣ 常见问题FAQ

### Q1：小moe不回复怎么办？

**检查清单**：
1. bot.py是否在运行？
   ```bash
   ps aux | grep bot.py
   ```
2. 是否@了小moe？（必须@才会回复）
3. 查看日志：
   ```bash
   tail -f bot_debug.log
   ```

**常见原因**：
- bot.py进程停止 → 重新启动
- 网络连接中断 → 重启机器人
- API限流 → 等待1分钟后重试

### Q2：知识库问答不准确怎么办？

**优化步骤**：
1. 检查系统提示词是否包含 `{{#context#}}`
2. 检查文档是否成功上传到AIME知识库
3. 在AIME平台查看知识库状态（绿色=就绪）
4. 重新运行`@小moe 更新知识库`

### Q3：产品查询找不到怎么办？

**可能原因**：
- 型号输入错误 → 检查拼写
- 产品不在库中 → 添加到 `product_library.json`
- 使用了不支持的查询方式 → 尝试完整型号

**支持的查询格式**：
```
✅ T8215
✅ 8215
✅ S330
✅ 门铃
✅ zhihuiping（拼音）
❌ t-8215（不支持连字符）
```

### Q4：如何添加新产品到产品库？

编辑 `product_library.json`：

```json
{
  "PN": "T8999",
  "产品名称": "新产品名称",
  "CR时间": "2026-06-01",
  "MSRP": "$199",
  "预估成本": "$70",
  "优先级": "P1"
}
```

重启bot.py生效。

### Q5：行业日报不推送怎么办？

**检查清单**：
1. `daily_news.py` 是否在运行？
2. 群ID是否正确配置？
3. 查看日志：
   ```bash
   tail -f daily_news.log
   ```

**手动测试**：
```bash
python3 daily_news.py
```

### Q6：Claude Code本地版连接不上飞书？

**常见问题**：
1. 未登录飞书MCP
   ```bash
   npx -y @larksuiteoapi/lark-mcp login -a YOUR_APP_ID -s YOUR_APP_SECRET
   ```

2. Token过期
   - 重新运行login命令
   - 检查 `~/.config/lark-mcp/` 目录下的配置

3. 权限不足
   - 确认飞书应用开通了相应权限
   - 检查 `docx:document` 权限

### Q7：怎么停止/重启机器人？

```bash
# 停止
pkill -9 -f "bot.py"
pkill -9 -f "daily_news.py"

# 重启
cd ~/Desktop/feishu_bot
nohup python3 bot.py > bot_debug.log 2>&1 &
nohup python3 daily_news.py > daily_news.log 2>&1 &

# 查看状态
ps aux | grep python3
```

### Q8：机器人回复太慢怎么办？

**可能原因**：
1. 网络延迟 → 检查网络连接
2. AIME服务负载高 → 等待片刻
3. 知识库太大 → 优化知识库结构

**优化建议**：
- 知识库控制在20篇文档以内
- 单个文档不超过10000字
- 定期清理过期文档

~/Desktop/feishu_bot/
