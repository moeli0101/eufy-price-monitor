## 🛡️ 多层保障机制

### 第1层：预防机制

**1. 依赖缓存**
```yaml
- name: Cache Playwright browsers
  uses: actions/cache@v3
```
✅ 避免重复下载Chromium（节省10-15分钟）

**2. 错误日志**
```yaml
python daily_price_refresh.py 2>&1 | tee scrape.log
```
✅ 记录所有输出，失败时可查看

**3. 数据验证**
```bash
# 检查文件大小
if [ "$size" -lt 100 ]; then
  echo "❌ 文件太小可能损坏"
  exit 1
fi
```
✅ 防止损坏数据被提交

**4. Git推送重试**
```bash
for i in 1 2 3; do
  git push && break
  git pull --rebase && sleep 5
done
```
✅ 自动解决临时冲突

---

### 第2层：检测机制

**1. 健康检查脚本**
```bash
python3 health_check.py
```
输出：
- ✅ 文件完整性
- ✅ JSON格式正确性
- ✅ 数据新鲜度（<2天）
- ✅ 产品数量（≥50）
- ✅ 详细统计信息

**2. 自动Issue创建**
```yaml
- name: Create issue on failure
  if: failure()
  uses: actions/github-script@v6
```
✅ 失败时自动创建GitHub Issue通知

**3. 日志上传**
```yaml
- name: Upload logs on failure
  if: failure()
  uses: actions/upload-artifact@v3
```
✅ 失败日志保存7天供分析
